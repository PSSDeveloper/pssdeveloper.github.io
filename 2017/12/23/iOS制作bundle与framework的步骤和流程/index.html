iOS制作bundle与framework的步骤和流程

由于需要把代码和资源封装起来给第三方使用，也就是制作SDK，所以会进行下面的打包操作。

制作Bundle包

Bundle里一般放置着各种资源文件，比如图片，storyboard，xib，音视频文件等。
具体制作步骤如下所示：

  • 选择XCode创建一个MacOS新项目，选择在Framework & Library中选择Bundle类型。如下图所示：

  • 因为选择的是macOS平台下的Bundle包，所以需要进行一些设置，在Build Phases -> Architectures -> Base SDK选择为iOS SDKs

  • 可选设置设置完平台版本之后，在Build Phases -> COMBINE_HIDPI_IMAGES设置为NO，经测试在高版本的XCode中，设置为YES也不会导致，打包出的framework中的png图片格式变为tiff。

  • 可选项作为资源包，仅仅需要编译就好，无需安装相关的配置信息，发现在高版本的XCode上，系统会默认设置skip install为YES，因此该步骤可做可不做，将skip install设置为YES,并删除Installation Directory下的目录.

  • 设置完成之后，即可进行编译(Commond B)打出Bundle包了。然后，show in finder即可找到对应的资源包。

注意打包的过程中设备要选择Generic iOS Devices，即可打出能在真机上跑的资源包

制作framework静态库

静态库的制作也很简单，步骤如下：

  • 打开XCode，选择iOS平台，Framework & Library下的Cocoa Touch Framework创建出一个新的Project
  • 现在开始进行一些项目设置，在Build Settings中进行设置。
    • 有以下几个设置项
      1. 在Linking里，将Dead Code Stripping设置为NO
      2. Link With Standard Libraries设置为NO
      3. 将Mach-O Type改为Static Librariy


设置完毕之后的Linking里的选项如下图，修改之后的单项为深黑色:

  • 然后依旧是在Build Setting下的Architectures下增加armv7s架构

  • 然后设置Build Ative Architectures Only为NO

  • 然后将要打包成Framework的代码文件拖拽到工程目录下，并在与项目名称相同的.h文件下把要暴露出去的头文件按照指定格式（引用framework里文件的格式）填写。

  • 然后回到Build Phases -> Headers将要暴露出去的头文件由project或者private拖拽到public中去。
  • 然后就可以进行编译工作了，分别选择真机和模拟器进行编译，会编译出对应模式下的framework，会有Debug和Release的真机和模拟器之分。
  • 然后需要将打包出的多个framework给合并成一个framework。如图所示，选中TARGETS -> Editor -> Add Target添加一个新的target到本target。
  • 然后选择Cross-platform跨平台这个target，选中Aggregate模版，然后按照提示创建出target。
  • 然后选中刚才创建的那个target，点击左上角的+号按钮，选择添加一个脚本文件，并将以下脚本代码添加进去。

  • 经过以上步骤，我们就可以制作出一个可以使用的framework了。需要注意的是，如果该framework包含有资源文件，需要在用到该framework的项目中，将该framewoirk加入到Copy Bundle Resources 中去，否则在使用过程中如果用到对应的资源会报错。
